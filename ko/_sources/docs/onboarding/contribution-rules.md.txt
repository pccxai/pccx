# 기여 규칙 (draft)

기여는 공개 기여물에 대한 기술 기여행위로 판단한다. `pccxai/pccx`,
`pccxai/pccx-v002`, `pccxai/pccx-FPGA-NPU-LLM-kv260` 각 저장소의
`CONTRIBUTING.md`가 브랜치 규칙, 커밋 형식, PR 흐름의 1차 권한 기준이다.

## 기여로 바로 얻는 것과 얻지 못하는 것

아래 항목은 기여만으로 자동 부여되지 않는다.

- 기여자에게 자본적 이익이 자동 발생하지 않음(지분, 배당, 수익분배 제외)
- 고용·고문 관계, 또는 상업적 정산 권한의 자동 부여
- 투자자 권리(의결권·우선매수권·배당권 등) 자동 부여
- 제품 구매가격, 라이선스, 계약에서 정한 혜택 외의 추가 상업 혜택

기여는 기여이며, 증권 거래·투자 제안·고용 계약이 아니다.

## CLA(임시 상태)

상용 사용 가능성 있는 레포의 코드는 CLA 발행 시점부터 공통적으로
`PCCX CLA`가 요구한다. 현재는 Apache-스타일 초안 기획 단계이며,
모든 기여는 기존 공개 라이선스로 우선 처리한다.
CLA가 발행되면 기존 기여에 대해 동의 재확인 절차를 실시하고,
동의하지 않는 기여는 기존 라이선스 범위에서만 취급한다.

## 기여자가 기대할 수 있는 점

- 기술 승인 기준은 저장소별 `CONTRIBUTING.md`에 따른다.
- 저자 정보는 `git log`에 유지되며, 임의로 삭제되지 않는다.
- 현재 라이선스 적용 시점에 대한 귀속은 기존 공개 라이선스 규칙을 따른다.
- 라이선스 변경은 CLA가 유효해진 이후 절차적으로 반영된다.

## 문의

CLA 발행 전의 논의는 저장소의 공식 `Discussions` 또는 이슈로 진행한다.
비공식 약속(개별 채팅/DM)보다 공식 문서와 정책이 우선이다.
