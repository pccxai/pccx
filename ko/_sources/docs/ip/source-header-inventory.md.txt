# Source-header inventory

`copyright-header-policy.md` 기준으로 PCCX 소스 헤더 반영 범위를 추적한다.

## 범위

다음 항목 보유가 대상:

- `PCCX(TM)` 또는 `PCCX™` 헤더 라인
- `SPDX-FileCopyrightText`
- `SPDX-License-Identifier`

다음은 제외:

- `.git`, `node_modules`, `dist`, `build`, `out`, `cache`, 가상환경
- `third_party`, `vendor`, submodule
- `reference_to_cheet`, `00_extracted_zips` 등 추출 보관소
- `codes/`(빌드 중 외부 RTL 클론 디렉터리)
- lock 파일: `package-lock.json`, `Cargo.lock`, `go.sum` 등
- `*.min.js`, `*.min.css`
- generated/do not edit/autogenerated 표기, 첫 30줄 기준
- binary(NUL 포함)

## 보고 출처

- 정책: `copyright-header-policy.md`
- 이 페이지: 커버리지 보고의 서사

## 산출물 생성

스크립트 실행시 JSON 리포트를 만들어 local tooling workspace에 보관한다
(레포지토리에 영구 저장하지 않음). 리포트에는 탐색 파일 수, 변경/건너뜀 수,
스킵 사유별 카운트가 들어간다.

## 왜 레포에 저장하지 않는가

실제 리포트는 실행 환경에 따라 바뀔 수 있어, 공개 저장소에는 상태 요약만 남기고
상세 리포트는 도구 워크스페이스에서 관리한다.
