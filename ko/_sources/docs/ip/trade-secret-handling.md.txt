# 영업비밀 처리 운영 지침

영업비밀은 공개성으로 가치가 사라지므로, 보호 범위를 명확히 구분해
운영한다. 영업비밀은 원칙적으로 다음 조건이 충족되어야 관리 대상이 된다.

- 외부에 공개되지 않은 기밀 정보일 것
- 재산적 가치가 공개 여부에 따라 급격히 달라질 것
- 접근·공유 범위를 계약과 내부 규정으로 통제할 수 있을 것

## 영업비밀로 분류되는 영역

- `pccx-v00N`의 ProCore 계열 RTL 묶음
- 폐쇄형 컴파일러 백엔드
- 성능·전력·면적 타이밍 튜닝 스크립트
- 고객별 최적화 소스와 통합 산출물
- 공개 특허 전환이 결정되지 않은 발명 후보

## 영업비밀에서 제외되는 항목

- `pccx`, `pccx-v00N`, 공식 공개 저장소에 올라간 오픈 코어
- 아키텍처 스펙, ISA reference, 공개 레지스터/메모리 맵, 공개 문서
- 시뮬레이션 wrapper, 공개된 형식의 evidence pack

## 접근 통제

영업비밀 항목은 공개 레포지토리 밖에서 별도의 기밀 통제 하에 관리한다.
접근은 다음 두 조건을 함께 만족해야 한다.

1. 직무 또는 협업 범위상 필요성이 입증될 것
2. 비밀유지 동의와 추적 규칙이 선행될 것

## 공개 안전 경계

- 영업비밀 원본, 패치, 회로별 최적화 로그는 공개 저장소에 두지 않는다.
- 공개 페이지에는 원칙, 범위, 점검 방식만 남기고 구현 상세는 공개하지 않는다.
- 분류가 바뀐 항목은 즉시 private docket/운영 기록에서 일괄 갱신한다.

## 상태

현재 공개 가능한 항목은 `docs/ip/trade-secret-policy.md`와
`private-disclosure-boundary.md`의 공개-safe 요건을 따르며, 본 문서는
운영 방식 개요만 제공한다.
