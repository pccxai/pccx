# Patent strategy

한국은 선출원주의입니다. 공개를 먼저 하면 후속 청구가 제한될 수 있어,
후보 발명은 분류 후 공개해야 한다. 본 페이지는 특정 발명을 공개하지 않으며
후보 분류 기준을 다룬다.

## 분류

각 후보 발명은 아래 하나로만 분류한다.

| 카테고리 | 처리 |
| --- | --- |
| Already publicly disclosed | 기존 공개 이후이므로 추가 출원 조치 없음 |
| Not public yet | 추가 공개 전 내부 결정 필요 |
| Patent candidate | 출원 의사결정 완료 후 선행 출원 필수 |
| Trade secret candidate | 특허보다 비공개 유지가 유리 |
| Defensive publication candidate | 경쟁자 선점을 막기 위한 공개 전략 |

## 공개 전 절차

1. 최소 신규 claim 정의
2. 분류 수행(불명확 시 not public yet)
3. patent candidate면 출원 선행
4. trade secret candidate면 비공개 유지
5. defensive publication candidate면 공개 시점·저자성 명시

## 후보 카테고리(비구체적)

- activation과 weight 경계에서의 quantization/arith pipeline
- KV-cache traffic 환경의 Transformer scheduling
- decoupled-dataflow를 위한 custom ISA encoding/decode
- compiler/runtime 기반 model graph와 IP-core instruction stream 맵핑
- single trace로 evidence를 재현하는 검증 허브

특정 발명명 공개는 private 프로젝트 절차를 통해만 진행한다.

## 상태

현재까지 PCCX는 등록 특허를 공식 발표하지 않았다. 공개 후보는 카테고리만
기재되며, 협업 이전에는 실제 발명을 열거하지 않는다.
