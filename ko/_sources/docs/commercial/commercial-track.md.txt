# Commercial track

Commercial track은 오픈 코어 위에 얹혀 제공되는 상용 계약형 산출물을 담는다.
오픈 라이선스만으로는 제공되지 않는 지원 항목과 의무를 고객이 받는다.

## 주요 제공 항목

| 항목 | 범위(의도) |
| --- | --- |
| **ProCore** | 특정 배포 시나리오에 맞춘 통합 준비형 RTL 번들. 상용 라이선스와 지원 의무를 따른다. |
| **Enterprise SDK** | 지불 고객 전용 컴파일러 백엔드, runtime, 통합 스크립트 체계. 비공개 빌드와 선행 접근권을 제공할 수 있다. |
| **ASICKit** | `tape-out` 대상 산출물. 파운드리 특화 RTL 뷰, 타이밍 스크립트, 검증 자료군 포함. |
| **Customer model porting** | 특정 고객 모델을 PCCX 타깃(보드 또는 향후 ASIC)으로 이식하는 엔지니어링 작업. 산출물 범위는 건별 SOW로 정의. |
| **Integration support** | 보드 bring-up, driver 통합, 검증 단계의 직접 지원. |
| **Certification** | 고객 통합이 PCCX 호환 조건을 충족했는지를 공식 점검하고 확인하는 절차. |

## 가격/조건

가격은 **TBD**입니다.. 공개 가격표는 없다. 비공식 채널의 개략 가격은
구속력이 없으며, 모든 상용 트랙 작업은 프로젝트 상용 법인과의 서면
계약, 적용 법규 하에서 진행된다.

## 오픈 트랙과의 경계

Commercial-track 고객은 오픈 트랙 산출물에도 기존 오픈 라이선스 기준으로
완전한 권리를 가진다. 상용 트랙은 오픈 사용권을 제약하지 않고 추가
지원·산출물만을 덧붙인다.

## Capital track과의 경계

Commercial-track 결제는 고객-공급자 거래다. 고객은 산출물에 대해 대가를
지불하고 프로젝트는 계약대로 전달한다. 이는 투자, 지분, 증권이 아니다.
투자 관계는 [capital track](capital-track.md)에 따로 서면 계약이 필요하다.

## 상태

현재 상용 계약은 유효하지 않으며, 계약 템플릿은 법무 검토 완료 후 별도 문서로
배포될 예정으로, 현재 저장소 내 본문에는 없다.
